Log

Log

These are used to keep record of events like bend program data, warning messages, error messages etc. Location where log data needs to stored can be changed by using 'Save Log' tab available in right side of screen in log tab.

General

Log events

Range

On / Off

Default

Off

Units

Access

Administrator

Function

This parameter specifies whether log data needs to be saved or not.

Description

Only if this parameter is ON other log level options will be enabled. If this parameter is OFF log events will not get saved.

Maximum days

Range

2 to 60

Default

10

Units

Days

Access

Administrator

Function

This parameter specifies the number days for which the log data will be available.

Description

By default, this parameter will be set to 10 days, meaning the log data of last 10 days will be available.

Time interval to log data

Range

1 to 10000

Default

500

Units

Second

Access

Administrator

Function

This parameter specifies the time interval with which the log data will be written.

Description

For every 30s, log data will be written to the file. Time interval range is from 1s to 10000s. Recommended value is 500s, which is default.

Log data is stored in location 'C:\users\<username>\AppData\Local\TJS'.

For every 500 lines of message created, data will be written to the file irrespective of time interval.

Log level

Information

Range

ON/OFF

Default

OFF

Units

Access

Administrator

Function

This parameter specifies whether information needs to be stored in log data.

Description

By default, this parameter is OFF, if the information needs to be stored in log data this parameter needs to be turned ON.

Bend program data

Range

ON/OFF

Default

OFF

Units

Access

Administrator

Function

This parameter specifies whether bend program data needs to be stored in log data.

Description

By default, this parameter is OFF, if the bend program data needs to be stored in log data this parameter needs to be turned ON.

Warning message

Range

ON/OFF

Default

OFF

Units

Access

Administrator

Function

This parameter specifies whether warning messages needs to be stored in log data.

Description

By default, this parameter is OFF, if the warning messages needs to be stored in log data this parameter needs to be turned ON.

Error message

Range

ON/OFF

Default

ON

Units

Access

Administrator

Function

This parameter specifies whether error messages needs to be stored in log data.

Description

By default, this parameter is ON, if the error message data need not to be stored in log data this parameter needs to be turned OFF.

Trace log

Range

ON/OFF

Default

OFF

Units

Access

Administrator

Function

This parameter specifies whether trace file needs to be stored in log data.

Description

By default, this parameter is OFF, if the trace file needs to be stored in log data this parameter needs to be turned ON.

Maximum number of trace file per day

Range

1 to 50

Default

10

Units

Access

Administrator

Function

This parameter specifies the number of trace file that needs to be logged per day.

Description

By default, this parameter is set to 10.
